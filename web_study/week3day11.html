9장 자바스크립트 기초 문법 살펴보기

9.1 변수와 상수

9.1.1 변수

변수는 변하는 수를 뜻한다. 자바스크립트에서 값이 변하는 데이터를 저장하고 관리하기 위한 공간으로 사용한다. 

                    
var num = 10 + 20;
해당 코드에서 var 는 키워드, num 은 식별자, = 는 연산자, 10 + 20 은 표현식, ; 은 세미콜론이다.

  • 키워드

    키워드란 자바스크립트 프로그래밍 언어에서 어떤 역할이나 기능이 정해진 특별한 단어이다. 다른 용어로 예약어라고도 한다.

  • 식별자

    식별자는 자바스크립트 내부에서 변수, 함수 등에 부여되는 이름을 의미한다. var, let, const 과 같은 변수를 생성하는 키워드와 함께 사용하면, 변수명이라고 한다.

  • 연산자

    연산자는 이름 그대로 어떠한 연산 작업을 하는데 사용하는 기호이다.

  • 표현식

    표현식이란 평가되어 하나의 값을 반환하는 식 또는 코드를 의미한다.

  • 값

    값은 더 이상 평가할 수 없는 데이터를 의미한다. 10 + 20 의 결과값은 30이 나오고 이를 더 이상 평가할 수 없기 떄문에 값이 된다.

  • 세미콜론

    세미콜론은 자바스크립트에서 하나의 명령어가 끝났음을 의미한다.

  • 선언, 할당, 초기화

    let, const, 키워드를 사용해 변수의 식별자를 지정하는 행위를 변수를 선언한다고 한다.
    할당 연산자인 = 기호로 우변에 있는 값을 변수 공간에 대입하는 것을 값을 할당한다고 한다.
    그리고, 선언과 할당을 한 번에 하면 변수를 초기화 한다고 한다.

9.1.2 새로운 변수 선언 키워드 let

let 키워드는 변수를 선언하는 새로운 키워드로, var 키워드보다 여러 기능이 개선된 키워드 이다.

  1. 변수명 중복이 불가능하다.

    var 키워드로 선언한 변수명은 중복해서 생성할 수 있다. 하지만, let 키워드는 변수명을 중복 선언하지 못하게 해서 개발자가 미처 중복된 변수를 사용하지 못하도록 한다.

  2. 호이스팅되지 않는다.

    호이스팅은 var 키워드로 변수를 선언하고 할당했을 때, 변수 선언을 자바스크립트의 스코프 맨 위로 올려 실행하는 것을 말한다. 

                                    
console.log(num);
var num = 10;
    이 코드를 사용하면, 에러가 떠야 하지만, 호이스팅에 의해 변수 선언이 맨 위에 올라가게 되고, 그 다음 값을 할당하기 때문에 출력하면 undefine 이 나온다.

    하지만, let 키워드는 호이스팅이 일어나지 않아서 num 변수를 정의하지 않았다면, 오류가 발생하게 된다.

  3. 스코프의 범위가 다르다.

    var 키워드와 let 키워드는 참조하는 스코프가 조금 다르다.

9.1.3 상수

const 키워드를 사용하여 상수 변수를 선언할 수 있다. 변수명을 중복으로 선언하는 것이 안되고, 호이스팅도 되지 않는다. 기본적으로 상수의 특징을 가지고 있어, 값을 재할당할 수 없다.

9.1.4 식별자 명명 규칙

식별자를 지정할 때는 몇 가지 규칙이 있다. 식별자에는 키워드를 사용할 수 없고, 공백을 포함할 수 없다.

관용적 식별자 명명 규칙도 있으며, 식별자는 영문으로만 작성해야 하고, 의미 있는 단어로 작성해야 한다.
그리고, 식별자를 표기하는 방법으로는, 카멜 표기볍, 언더스코어 표기법, 파스칼 표기법이 있다.

9.2 자료형

자료형이란 자바스크립트에서 사용할 수 있는 데이터의 종류를 의미한다. 자바스크립트의 자료형은 기본 자료형과 참조 자료형으로 구분한다.

9.2.1 문자열

문자열은 큰따옴표나 작은따옴표로 둘러싸인 값의 형태를 말한다. 문자열에 따옴표가 포함된 경우, 문자열에 포함되지 않은 따옴표로 문자열을 감싸서 정의하면 된다.

  • 문자열 연결 연산자

    문자열 연결 연산자는 덧셈 기호로 표현하며, 문자열과 문자열을 덧셈 기호로 연산하면 서로 연결이 된다.

  • 이스케이프 문자

    이스케이프 문자는 사용자의 의도와 다르게 문자열을 해석할 때 사용한다. 보통 문자열 정의로 사용하는 것이 아닌 순수한 문자임을 사용하고 싶을 때 사용한다.
    역슬래시를 붙여 이스케이프 문자를 사용하면 된다.

  • 템플릿 문자열

    템플릿 문자열은 백틱으로 문자열을 정의하는 방법이다. 큰따옴표와 작은따옴표를 사용하지 않았기에 문자열에 두개 모두 있어도 영향을 받지 않는다.
    또한, 엔터를 입력해서 줄바꿈을 하면 적용되고, ${} 문법을 사용해 문자열에 변수 또는 식을 넣을 수 있게 한다.

9.2.2 숫자형

숫자형은 정수와 실수 같은 숫자 자료형을 의미한다. 실수를 더할 때는 부동소수점에 의해 실수 계산이 정확하지 않다는 것을 주의해야 한다.

9.2.3 논리형

논리 자료형은 참 또는 거짓에 해당하는 논리 값인 true, false 를 의미한다. 값을 직접 지정할 수 있고, 논리적인 연산으로도 구할 수 있다.

9.2.4 undefined

변수나 상수를 메모리 공간에 선언하면 반드시 생성한 공간에 저장할 데이터를 할당해야 한다. 만약, 할당하지 않을 경우 자바스크립트 내부에서 임의로 undefined 값을 할당한다.

9.2.5 null

null 자료형은 null 값 하나만 있으며, 변수나 상수를 선언하고 의도적으로 선언한 공간을 비워 둘때 할당한다.

9.2.6 객체

객체는 자바스크립트의 핵심적인 자료형으로 배열, 객체 리터럴, 함수가 있다.

  • 배열

    배열은 복수의 데이터를 정의할 수 있는 자료형이다. 배열 요소에 접근하려면 인덱스를 이용해야 하며, 인덱스는 배열에서 각 데이터가 있는 위치를 가리키는 숫자이다.
    배열이 할당된 변수명에 대괄호를 사용해 인덱스 숫자르 넣으면, 위치의 데이터에 접근할 수 있다.

  • 객체 리터럴

    객체 리터럴은 객체를 정의할 때 중괄호를 사용하며, 중괄호 안에는 키와 값의 한쌍으로 이루어진 속성이 들어간다. 배열과 다르게 값만 사용하는 것이 아닌 키를 사용하기 때문에, 배열에 담긴 값들이 어떤 값인지 명확하게 알 수 있다.

    값에 접근할때는 대괄호를 사용하거나, 마침표를 사용해 변수명과 키값을 적어 접근한다.

9.3 연산자

연산자는 어떤 연산을 저리하는 데 사용하도록 미리 정의된 기호를 말한다.

9.3.1 산술 연산자

  • 이항 산술 연산자

    이상 산술 연산자는 연산을 수행하는 데 피연산자가 2개 필요한 연산자이다. 일반적으로 수학에서의 산술 연산과 같다.

  • 단항 산술 연산자

    단항 산술 연산자는 연산을 수행하는 데 피연산자가 1개만 필요한 연산자를 의미한다. ++연산자는 값을 1 증가시키고, -- 연산자는 값을 1 감소시킨다.
    앞에 사용하면 전치연산, 뒤에 사용하면 후치 연산이라고 한다. 전치연산은 값을 할당하기 전에 1을 증가시키고, 후치연산은 값을 할당한 후에 1을 증가시킨다.

  • 단항 부정 연산자

    단항 부정 연산자는 항상 피연산자 앞에 위치하며 피연산자의 부호를 부정하는 연산을 부행한다. 음수는 양수로 양수는 음수로 변환한다.

9.3.2 대입 연산자와 복합 대입 연산자

대입 연산자는 데이터를 대입하는 연산을 수행하는 연산자를 말한다. 복합 대입 연산자는 산술 연산자와 대입 연산자를 함께 사용해 산술과 할당을 한 번에 수행하는 연산자를 말한다.

9.3.3 비교 연산자

비교 연산자는 피연산자를 비교한 뒤, 논리형 값인 참, 거짓을 반환하는 연산을 수행한다.

9.3.4 논리 연산자

논리 연산자는 피연산자를 논리적으로 평가한 뒤, 조건에 맞는 피연산자를 반환하는 연산을 수행한다.
"", undefined, 0, null 는 거짓으로 평가되고, 나머지는 참으로 평가된다. AND 연산자는 연산 결과 거짓으로 평가되면 거짓으로 평가된 피연산자를 반환한다.
거짓이 아니면, 마지막에 평가되는 피연산자를 반환한다.

OR 연산자는 피연산자를 왼쪽부터 평가해 참으로 평가된 피연산자를 즉시 반환한다. 만약 모든 피연산자가 참으로 평가되지 않으면 마지막에 평가된 피연산자를 반환한다.

NOT 연산자는 피연산자나 식을 평가한 논리 값의 반대값을 반환한다.

9.3.5 삼항 연산자

삼항 연산자는 세 항 중 가장 왼쪽에 있는 피연산자의 참, 거짓에 따라 나머지 두 항에 있는 피연산자를 선택적으로 반환하는 연산을 수행한다. 

                    
let score = 90;
let grade = score >= 90 ? 'A' : 'B';
console.log(grade); // A+
이를 사용하면 조건ㅁ누을 사용하지 않아도 간단한 조건식을 처리할 수 있다.

9.3.6 연산자 우선순위

연산자 우선순위는 연산자를 여러 개 사용했을 때 어떤 연산자를 먼저 연산할지를 결정하는 기준이다. 우선순위가 같으면 결합 순서에 따라 연산이 수행된다.

9.3.7 형 변환

형 변환은 데이터의 자료형이 다른 자료형으로 바뀌는 것을 말한다.

  • 암시적 형 변환

    암시적 형 변환은 사용자가 형 변환을 의도하지 않았지만, 자바스크립트에서 자체적으로 형 변환하는 것을 말한다.

  • 명시적 형 변환

    명시적 형 변환은 형 명시적으로 형 변환을 처리하는 것을 말한다. 형변환을 하는 메서드를 사용해서, 명시적 형 변환을 진행한다.
