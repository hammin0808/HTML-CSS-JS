11장 자바스크립트 객체 다루기

11.1 객체란

객체는 키와 값으로 구성된 속성의 집합을 의미한다. {} 를 이용해 생성할 수 있으며, 이러한 방법을 리터럴 방식으로 객체를 생성했다고 한다.
객체는 데이터의 종류를 가리지 않아 모든 자료형의 데이터를 값으로 가진다. 예를 들어, 배열, 숫자, 논리 데이터를 한 객체의 속성으로 추가할 수 있다.
또한 객체 안에 객체나 함수가 들어갈 수도 있다.

11.2 객체 속성 다루기

11.2.1 객체 속성에 접근하기

  • 대괄호 연산자로 접근하기

    [] 를 사용해 객체의 속성에 접근하는 방법과 배열에서도 사용할 수 있다. 객체의 속성에 접근하려면 객체명 뒤에 대괄호를 붙이고 키를 넣으면 된다.

    객체 리터럴 값에 접근하려면, 대괄호 연산자를 연속으로 사용해 접근하고, 배열 값에 접근하려면, 배열의 인덱스 인자를 그 뒤에 붙여 접근하면 된다. 함수를 호출할 때는 그 뒤에 소괄호를 사용하면 된다.

  • 마침표 연산자

    . 를 사용해 객체 속성에 접근한다. 이때 대괄호 연산자는 큰따옴표나 작은따옴표로 객체의 속성에 접근하는 것과 달리 키 값을 바로 적으면 된다. 객체의 속성 사이에 공백이 있을 경우 대괄호 연산자를 사용해 접근한다.

11.2.2 객체 속성 값 변경하기

객체로 정의된 값을 바꾸고 싶으면 키로 속성에 접근해서 값을 재할당하면 된다.

11.2.3 객체 속성 동적으로 추가하기

객체 속성에 키로 접근해 값을 재할당 하면 기존 속성값을 변경할 수 있지만, 해당 키가 객체에 없다면 해당 키와 값으로 구성된 새로운 속성이 객체에 추가된다.
객체 속성의 값이 함수나, 배열, 객체 리터럴 일때도 마찬가지로 없는 속성이면 해당 키와 값으로 구성된 새로운 속성을 객체에 추가한다.

11.2.4 객체 속성 동적으로 삭제하기

delete 키워드로 객체 속성을 동적으로 삭제할 수 있다.

11.2.5 객체의 데이터 관리 방법 이해하기

객체의 생성은 const 키워드로 생성한 반면 객체 내부의 값을 자유롭게 변경할 수 있었다. 이는, 객체 자료형의 특성인 참조때문이다.

  • 기본 자료형의 데이터 관리 : 깊은 복사

    기본 자료형은 변수에 데이터를 할당할 때 데이터 그 자체가 할당된다. 이 상태에서 변수에 할당된 데이터를 다른 변수에 다시 할당하면 데이터를 복사해 전달한다.
    기존 변수에 담겨있던 데이터 값을 변경해도 다른 변수의 데이터 값은 연동되어 있지 않아 변경되지 않는다. 이를 깊은 복사라고 칭한다.

    깊은 복사 : 복사한 값을 재할당할 때 한쪽 데이터가 변경되어도 서로 영향을 미치지 않게 복사되는 것

  • 참조 자료형의 데이터 관리 : 얇은 복사

    참조 자료형은 변수 공간에 데이터가 할당되는 것이 아니고, 데이터가 위치하고 있는 메모리의 주소 값만 할당된다. const 키워드로 생성한 객체의 변수에 다른 객체를 재할당하려고 하면 오류가 발생한다.
    하지만 변수에 할당된 객체에 속성을 추가하거나 값을 변경하는 것은 가능하다. 이는 재할당하는 것이 아닌 주소 값이 참조하는 원본 객체 데이터만 변경되는 것이기 때문이다.
    이러한 특징은 데이터를 복사했을 때 주소 값만 복사하기 때문에 원본 데이터를 변경하면 다른 변수 값도 해당 데이터로 변경된다. 이를 얕은 복사라고 한다.

    얕은 복사 : 데이터를 복사했을 때 한쪽 데이터가 변경되면 다른 쪽 데이터도 변경되어 서로 영향을 받는 것
